# AI 语义问答与维护规则

## 1. 首次使用

1. 安装并启动 Ollama。
2. 在设置的“本地 AI 与语义索引”中填写本地服务地址，默认是 `http://127.0.0.1:11434`。
3. 点击“检测连接”，选择一个 Embedding 模型和一个对话模型。
4. 点击“保存配置”。首次保存会在后台建立分块索引，阅读和编辑不会被锁定。

如果 Ollama 未安装或模型不可用，AI 侧栏仍可使用关键词检索，不会影响普通搜索、编辑、图谱和文档保存。

## 2. 语义问答

打开顶部的 `AI` 侧栏后，可以选择“全知识库”或“当前文档”。回答下方的来源卡片包含文档名、标题和行号，点击后会打开原文并定位到对应标题。

回答只应依据来源卡片中的内容。没有足够依据时，系统会提示补充问题，不应把模型猜测当作知识库事实。

## 3. Frontmatter 标准

新文档使用以下字段：

```yaml
---
schema: mytemple/v1
title: 文档标题
tags:
  - 标签
domain: 所属领域
created: 2026-07-22
updated: 2026-07-22
status: draft
aliases: []
---
```

旧文档可在设置中点击“标准化当前文档”。应用前会展示差异、检查文档是否被其他程序修改，并在本地创建备份。

## 4. AI 规则文件

点击设置中的“创建规则文件”后，工作区会生成 `.mytemple/AGENTS.md`。默认规则是 `writeMode: confirm`：AI 可以提出创建、补丁修改或标签建议，但写入前必须展示差异并由用户确认。

规则文件中的正文只是行为约束，不能授予 AI Shell 权限，也不能覆盖应用的路径安全、敏感文件保护和冲突检查。

## 5. 索引维护

- 修改或新增文档后，只会重新处理发生变化的分块。
- 删除文档后，其分块会从索引中移除。
- 更换 Embedding 模型后需要重建索引。
- 索引文件保存在用户数据目录，不写入项目安装目录。
