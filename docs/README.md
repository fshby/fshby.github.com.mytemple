# MyTemple Knowledge 使用手册

MyTemple Knowledge 是一个以本地 Markdown 为核心的文档知识库。文档仍然保存在普通文件夹中，应用负责阅读、编辑、检索、标签整理和知识神经网络展示。

## 功能目录

### 文档使用

- 在左侧目录切换工作区、文件夹和文档。
- 使用顶部搜索框检索标题、路径和正文内容。
- 在阅读模式查看正文目录，三级目录默认折叠在二级目录下。
- 在修改模式编辑 Markdown、实时预览并自动保存。
- 使用沉浸模式隐藏非必要导航，专注于当前文档。

### 知识整理

- [知识神经网络与文档标记指南](知识神经网络与文档标记指南.md)
  - 图谱节点和连接线代表什么；
  - 标签、语义概念和文档双链的区别；
  - 全局图谱、当前文档邻域与领域深度的使用方法；
  - 怎样编写能生成清晰知识脉络的文档。
- [文档编写与管理规范](文档编写与管理规范.md)
  - 文件夹和文件命名；
  - 标准文档模板；
  - 标题、标签、双链和状态的写法；
  - 日常收集、整理和归档流程。

### 辅助工具

- “智能适配标签”可以分析旧文档并预览建议标签。
- “一键 MD 化”可以统一可转换文件的扩展名。
- 设置中可以切换主题、字体、字号和默认工作路径。
- 设置中的沟通群入口提供安装包内置的群说明和二维码，只读展示且不可在界面修改。

### AI 快速索引

- 应用每次刷新文档知识库时，会同步生成 [`知识库主清单.json`](知识库主清单.json)。
- 主清单包含工作区、文档标题、路径、标签、目录标题、摘要、语义概念、双链、缺失链接以及图谱节点和边。
- AI 工具应先读取主清单筛选候选文档，再通过其中的 `absolutePath` 按需读取原文，避免一次加载整个知识库。
- 字段定义、刷新机制和使用方法参见[知识库主清单说明](知识库主清单说明.md)。

## 推荐起步方式

1. 先阅读[文档编写与管理规范](文档编写与管理规范.md)，确定文件夹和标签规则。
2. 为常用文档补充标准 frontmatter 和至少一个明确双链。
3. 打开[知识神经网络与文档标记指南](知识神经网络与文档标记指南.md)，学习如何筛选图谱关系。
4. 每周检查一次孤立文档、未创建链接和“待分类”标签。
