# 知识库主清单说明

`知识库主清单.json` 是提供给本地 AI 工具和大模型的轻量知识库入口。它随文档知识库刷新自动重建，位置固定在应用安装目录的 `docs` 文件夹中。

## 设计目的

- 让 AI 先读取一个结构化索引，而不是一次加载全部 Markdown 原文。
- 同时暴露标题、标签、目录层级、摘要、关键词、语义概念、双链和图谱关系。
- 使用文档内容哈希和修改时间判断候选文档是否发生变化。
- 保留原文绝对路径，命中候选后可以按需读取少量原文。

## 主要字段

- `library`：工作区、文档、节点、边、标签、语义概念和缺失链接数量。
- `workspaces`：当前参与刷新和图谱生成的工作区清单。
- `documents`：文档路径、标题、标签、标题层级、关键词、摘要、双链、反向链接和内容哈希。
- `graph.nodes`：文档、显式标签、语义概念与未创建链接节点。
- `graph.edges`：双链、标签、语义和未创建链接关系。

## 推荐 AI 读取流程

1. 读取 `知识库主清单.json`。
2. 根据 `documents[].title`、`tags`、`headings`、`keywords` 和 `excerpt` 筛选候选文档。
3. 使用 `outgoingLinks`、`backlinks`、`semanticConcepts` 和 `graph` 扩展关联文档。
4. 最后读取候选项的 `absolutePath` 原文并生成回答。

## 刷新规则

- 手动刷新知识库、增删或移动文档、切换工作区后立即同步。
- 编辑保存采用短延迟合并更新，避免连续输入时频繁写入磁盘。
- 主清单自身不会进入文档树和知识图谱，防止索引递归索引自己。
