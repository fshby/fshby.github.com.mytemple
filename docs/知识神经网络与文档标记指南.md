# 知识神经网络与文档标记指南

知识神经网络不是把所有标题堆成标签，而是把文档、显式标签、语义概念和文档链接组织成可探索的关系网络。标记越准确，网络主干越清晰。

## 一、认识图谱元素

### 文档节点

文档节点代表一篇本地 Markdown 文档。点击文档节点可以打开正文，节点连接数越高，说明它与其他知识的联系越多。

### 显式标签节点

显式标签来自 frontmatter 的 `tags` 字段或正文中的 `#标签`。它表达作者主动确认的分类关系，适合表示稳定主题、领域和用途。

```md
---
tags:
  - 网络运维
  - 故障处理
---
```

也可以在正文末尾使用：

```md
#网络运维 #故障处理
```

### 语义概念节点

语义概念由系统根据全库词频和文档内容自动提取。它用来发现尚未手工标记的潜在关系，不等于显式标签，也不会把所有 Markdown 标题识别成标签。

### 文档双链

双链表示作者明确建立的文档关系，通常是最强、最容易解释的知识连接。

```md
[[网络故障排查]]
```

也可以使用标准 Markdown 链接：

```md
[网络故障排查](运维/网络故障排查.md)
```

### 未创建链接

文档中存在 `[[目标文档]]`，但对应文档还未创建时，图谱会显示未创建节点。它可以作为后续写作清单使用。

## 二、标题不是标签

以下写法是 Markdown 标题，只负责正文结构：

```md
# 一级标题
## 二级标题
### 三级标题
```

以下写法才是正文显式标签，`#` 后没有空格：

```md
#项目管理 #需求分析
```

建议优先把稳定标签写入 frontmatter，正文 `#标签` 只用于临时补充。不要为了让图谱“看起来丰富”而把每个标题都复制成标签。

## 三、图谱操作方法

### 全局图谱

全局图谱展示当前可见工作区的全部关系，因此深度显示为“全部层级”。适合观察知识库整体结构、核心节点和孤立区域。

### 当前文档邻域

当前文档邻域以已经打开的文档为中心，只展示指定深度内可到达的节点。

- `1 层`：当前文档和直接关联节点；
- `2 层`：增加关联节点的直接邻居，适合日常探索；
- `3 层`：展示更广的上下文，适合分析跨领域关系。

在“全部层级”下选择 1～3 层，系统会自动切换到当前文档邻域。

### 关系筛选

- 只勾选“显式标签”：检查人工分类是否清晰；
- 只勾选“语义概念”：发现内容相似但尚未建立链接的文档；
- 同时关闭两项：重点查看文档双链；
- 勾选“孤立”：查找没有有效关系的文档；
- 勾选“未创建”：查看计划编写但尚未建立的文档。

### 浏览与拖动

- 拖动画布空白处：平移视图；
- 滚轮或 `+ / -`：缩放图谱；
- 点击“适配”：让全部可见节点回到视野内；
- 拖动节点：观察一至三阶关联节点的牵引和回弹；
- 点击文档节点：打开对应文档；
- 搜索节点：高亮匹配节点及其局部关系。

## 四、让知识脉络更清晰

### 每篇文档只表达一个主要主题

如果一篇文档同时包含多个无关主题，语义概念会变得分散。建议拆成多篇单主题文档，并通过双链连接。

### 每篇文档使用二至五个稳定标签

标签过少无法形成分类，标签过多会制造大量交叉线。优先使用：

- 领域标签：如 `网络运维`、`产品设计`；
- 内容类型：如 `操作手册`、`决策记录`；
- 生命周期：如 `进行中`、`已归档`。

不要把日期、完整句子、一次性编号和常见虚词作为标签。

### 使用双链表达真正的上下游关系

标签回答“它属于什么”，双链回答“它具体关联什么”。建议在正文中设置“关联文档”章节，记录前置资料、执行方案和结果文档。

### 为核心主题建立索引文档

索引文档也称主题地图或 MOC，用于串联一个领域的重要内容。

```md
# 网络运维主题索引

## 基础资料

- [[网络设备清单]]
- [[IP 地址规划]]

## 操作流程

- [[网络故障排查]]
- [[变更实施流程]]

## 复盘记录

- [[重大故障复盘]]
```

## 五、旧文档整理方法

1. 在设置中打开“智能适配标签”。
2. 先预览建议，不要直接接受所有标签。
3. 删除过于宽泛、含义重复或只出现一次的标签。
4. 为核心文档补充明确双链。
5. 在图谱中只显示标签，检查分类主干。
6. 再只显示语义概念，检查遗漏关系。
7. 最后查看孤立节点和未创建链接。

```md

  - 文档类型
  - 当前状态

# 清晰、唯一的文档标题

> 用一至两句话说明本文解决什么问题。

## 背景

说明来源、范围和约束。

## 核心内容

记录可复用的事实、方法或结论。

## 关联文档

- [[前置资料]]
- [[后续任务]]

## 更新记录

- 2026-07-21：创建文档。
```

## 七、常见问题

### 为什么图谱很乱

通常是标签数量过多、概念重复、文档主题不单一，或者缺少明确双链。先切换成单一关系模式，再逐步清理。

### 为什么邻域深度不可用

全局图谱没有中心节点，因此显示“全部层级”。打开一篇文档并选择 1～3 层后，会自动进入当前文档邻域。

### 为什么标题没有成为标签

这是预期行为。标题负责正文结构，只有 frontmatter `tags` 和正文 `#标签` 才是显式标签。

### 标签和语义概念应该选哪个

人工维护的稳定分类使用标签；系统发现的潜在内容关系使用语义概念。需要查看清晰主干时建议分别查看。
